1. Introducción

La Plataforma de Gestión de Comités Juveniles IMJUVE es un sistema web que permite registrar, consultar, editar y gestionar los Comités Juveniles por la Transformación a nivel nacional.

El sistema cuenta con control de acceso basado en roles (RBAC), dashboard interactivo con mapas y gráficas, generación automatizada de actas en PDF, y un módulo de auditoría que registra todas las acciones realizadas.

Funcionalidades principales

  • Registro de comités con captura de fotografía, datos del comité e integrantes
  • Generación de actas en PDF con código QR y diseño oficial IMJUVE
  • Dashboard con estadísticas, mapa de calor y gráficas dinámicas
  • Consulta y edición de comités registrados
  • Importación masiva desde archivo CSV
  • Gestión de usuarios con 4 roles distintos
  • Bitácora de auditoría exportable

2. Requisitos del Sistema

Para ejecutar el servidor

  • Node.js versión 18 o superior
  • Sistema operativo: Windows 10+, Ubuntu 20.04+, macOS 12+
  • RAM: mínimo 512 MB libres (recomendado 1 GB)
  • Almacenamiento: ~200 MB para la aplicación + espacio para fotos

Para acceder desde el navegador

  • Google Chrome 90+, Mozilla Firefox 90+, Microsoft Edge 90+, Safari 15+
  • Conexión a internet o red local
  • Resolución mínima recomendada: 1024×768

3. Instalación Local

Windows

  1. Instalar Node.js 18+ desde nodejs.org
  2. Copiar la carpeta del proyecto a la ubicación deseada
  3. Ejecutar iniciar-servidor.bat haciendo doble clic
  4. El script instalará dependencias, sembrará datos de prueba e iniciará el servidor
  5. Abrir el navegador en http://localhost:3001
En Windows, si PowerShell bloquea la ejecución de scripts, ejecute el Comando como Administrador: Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

Linux / macOS

  1. Instalar Node.js 18+ (Linux: sudo apt install nodejs npm)
  2. Abrir terminal en la carpeta del proyecto
  3. Ejecutar: chmod +x start-server.sh && ./start-server.sh
  4. Abrir el navegador en http://localhost:3001

Instalación manual paso a paso

  1. cd server
  2. npm install
  3. npx tsx src/seed.ts (opcional, crea datos de prueba)
  4. PORT=3001 npx tsx src/index.ts
  5. Abrir http://localhost:3001

4.1 Administrador

Acceso total y sin restricciones a toda la plataforma:

  • Registro, edición y eliminación de comités en cualquier estado
  • Dashboard con estadísticas nacionales
  • Estadística del Estado (completa)
  • Cargar Comités desde CSV
  • Administración de usuarios (CRUD completo)
  • Auditoría con bitácora exportable
  • Botón "Borrar todo" en Historial de Actas

4.2 Territorial

  • Puede registrar comités en cualquier estado
  • Puede editar y eliminar comités solo en sus estados asignados
  • Acceso a Dashboard y Estadística del Estado
  • No tiene acceso a Administración de usuarios ni Cargar Comités

4.3 Coordinador

  • Solo puede registrar comités en su estado asignado (1 estado)
  • Puede editar comités y campos de integrantes, pero no puede eliminar integrantes
  • Acceso a Consulta de Comités
  • Sin acceso a Dashboard, Estadísticas ni Administración

4.4 Promotor

  • Solo puede registrar comités en su estado asignado (1 estado)
  • Solo ve y descarga actas que él mismo registró
  • Sin acceso a Consulta, Dashboard, Estadísticas, Cargar Comités ni Administración

5. Módulos del Sistema

5.1 Registro de Comité

Formulario completo para registrar un nuevo comité.

Campos del formulario

  • Estado y Municipio: selección geográfica del comité
  • Fecha de protesta: fecha de toma de protesta
  • Eje temático: eje principal del comité
  • Ruta de articulación: Educativo o Territorial
  • Fotografía: foto del comité (opcional, se comprime automáticamente)
  • Integrantes: mínimo 5, cada uno con cargo, nombre, sexo, edad, municipio, teléfono, email y población vulnerable

Al guardar, el sistema genera automáticamente un folio único y un PDF del acta que puede descargarse.

Para roles con 1 estado asignado (coordinador, promotor), el estado se preselecciona automáticamente.

5.2 Historial de Actas

Lista de todas las actas registradas con opciones de:

  • Filtros: por estado y texto de búsqueda
  • Vista previa: previsualización del acta sin descargar
  • Descarga individual: descarga el PDF de un acta específica
  • Descarga masiva: descarga múltiples actas en un archivo ZIP
  • Eliminar: solo visible para administrador y territorial (en sus estados)
  • Borrar todo: solo visible para administrador, con doble confirmación + contraseña "CONTRASEÑA"

5.3 Consulta de Comités

Tabla con todos los comités registrados. Permite:

  • Editar inline: modificar datos del comité e integrantes directamente en la página (sin modal overlay)
  • Filtros: por estado, folio, nombre del comité

5.4 Dashboard

Panel de estadísticas con:

  • KPIs: Comités registrados, Integrantes totales, Estados activos, Promedio de integrantes
  • Mapa de México: calor por estado + desglose numérico en grid 4 columnas
  • Top 10 estados: ranking con barras de progreso
  • Registros por mes: gráfica de área (totales mensuales) + línea negra (promedio por día)
  • Distribución por edad: gráfica de área por rango etario
  • Distribución por sexo: gráfico de dona (Mujeres / Hombres)
  • Tamaño de comités: barras por rango de integrantes
  • Ejes Temáticos: barras por eje
  • Ruta de Articulación: tarjeta separada

5.5 Estadística del Estado

Edición de datos estadísticos por estado: población joven, municipios, partido gobernante, matrícula educativa y participación en jornadas.

5.6 Cargar Comités

Importación masiva de comités desde archivo CSV. Cada fila representa un integrante; los integrantes se agrupan automáticamente por nombre del comité. Mínimo 5 integrantes por grupo.

Ver sección Importación CSV para el formato detallado.

5.7 Administración

Panel con dos subpestañas:

  • Usuarios: CRUD completo de usuarios del sistema, importación desde CSV y descarga de plantilla
  • Auditoría: bitácora de todas las acciones realizadas, con filtros y exportación a CSV ("Descargar Bitácora")

7. Gestión de Usuarios

Accesible desde el menú Usuarios (solo administradores).

Crear usuario

  1. Hacer clic en "+ Nuevo usuario"
  2. Completar: nombre de usuario, contraseña, nombre completo, rol y estados asignados
  3. Hacer clic en "Guardar"

Editar usuario

  1. Hacer clic en el icono de lápiz junto al usuario
  2. Modificar los campos necesarios
  3. Hacer clic en "Guardar"

Eliminar usuario

  1. Hacer clic en el icono de papelera junto al usuario
  2. Confirmar la eliminación en el diálogo

Importar usuarios desde CSV

  1. Descargar la plantilla con el botón "Descargar Plantilla"
  2. La plantilla incluye ejemplos de los 4 roles
  3. Completar el archivo y usar "Importar CSV"

Formato de la plantilla:

username,password,role,full_name,assigned_states
ADMIN_M,0000,administrador,María López,[]
TERRI_E,0000,territorial,Eduardo Cruz,"[""JALISCO"",""NAYARIT""]"
COORD_N,0000,coordinador,Nancy Ríos,"[""NUEVO LEON""]"
PROMO_X,0000,promotor,Ximena Paz,"[""VERACRUZ""]"

8. Auditoría

El sistema registra automáticamente cada acción relevante en la bitácora de auditoría:

  • Creación, edición y eliminación de comités
  • Inicio de sesión de usuarios
  • Creación, edición y eliminación de usuarios
  • Acciones masivas (Borrar todo, importar CSV)

Para exportar la bitácora, usar el botón "Descargar Bitácora" en la pestaña Auditoría del panel de Administración.

9. Base de Datos

El sistema utiliza SQLite como motor de base de datos. El archivo se encuentra en:

server/data/imjuve.db

Estructura de tablas

Las tablas se crean automáticamente al iniciar el servidor por primera vez:

  • users — Usuarios del sistema (id, username, password_hash, role, assigned_states, full_name)
  • comites — Comités registrados (folio, estado, municipio, fecha_protesta, etc.)
  • integrantes — Integrantes de cada comité
  • audit_log — Bitácora de acciones
  • estado_estadisticas — Datos estadísticos por estado

Precargar datos de prueba

Para insertar 8 usuarios de prueba y 32 comités de ejemplo:

cd server
npx tsx src/seed.ts

Empezar desde cero

Eliminar el archivo server/data/imjuve.db y reiniciar el servidor. Las tablas se recrearán automáticamente.

Eliminar la base de datos borrará permanentemente todos los registros. Haga una copia de seguridad antes.

10. Despliegue en Producción

10.1 Servidor VPS con Nginx

Paso 1: Preparar el servidor

sudo apt update && sudo apt upgrade -y
sudo apt install nginx nodejs npm -y
node -v  # Debe mostrar v18+

Paso 2: Copiar la aplicación

scp -r PlataformaComites usuario@tu-servidor:/home/usuario/
# o clonar desde git
git clone https://tu-repo.com/plataforma-comites.git
cd PlataformaComites

Paso 3: Construir el frontend

cd client
npm install
npx vite build
cd ..

Paso 4: Inicializar base de datos

cd server
npm install
npx tsx src/seed.ts  # Datos de prueba (saltar si producción real)

Paso 5: Instalar PM2 y arrancar

sudo npm install -g pm2
pm2 start --name imjuve --interpreter tsx src/index.ts
pm2 save
sudo pm2 startup  # Arranque automático al reiniciar

Paso 6: Configurar Nginx

sudo nano /etc/nginx/sites-available/imjuve

Agregar:

server {
    listen 80;
    server_name tu-dominio.com;

    client_max_body_size 50M;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
sudo ln -s /etc/nginx/sites-available/imjuve /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

10.2 Plataforma como Servicio (Railway, Render, Fly.io)

Ejemplo con Railway:

  1. Crear cuenta en railway.app
  2. Conectar repositorio de GitHub o subir archivos
  3. En Settings → Deploy configurar:
  4. Build Command: cd client && npm install && npx vite build && cd ../server && npm install
  5. Start Command: cd server && npx tsx src/index.ts
  6. Root Directory: dejar vacío
  7. Agregar variable de entorno: JWT_SECRET=clave-secreta-aleatoria
  8. Hacer clic en Deploy

10.3 Configurar SSL (HTTPS)

Con Certbot y Let's Encrypt:

sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d tu-dominio.com

Certbot modificará automáticamente la configuración de Nginx para usar HTTPS.

Para renovación automática:

sudo certbot renew --dry-run

11. Solución de Problemas

El servidor no arranca

ProblemaSolución
Puerto 3001 en usoCambiar puerto: PORT=3002 npx tsx src/index.ts
"No se puede cargar el archivo" en PowerShellEjecutar: Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
Error de módulo no encontradoEjecutar: cd server && npm install

La página no carga en el navegador

ProblemaSolución
Puerto incorrectoAbrir http://localhost:3001 (no 5173 ni 5174)
Cache del navegadorCtrl+Shift+R (recarga forzada) o limpiar caché
Firewall bloqueaPermitir Node.js en el firewall o usar 127.0.0.1:3001

Error al iniciar sesión

ErrorSolución
"Usuario o contraseña incorrectos"Verificar que el usuario existe (ejecutar seed si es necesario)
Token inválidoCerrar sesión y volver a iniciar, o recargar la página

Problemas con roles

SíntomaSolución
Un usuario no ve las opciones de su rolRecargar página (Ctrl+Shift+R) para refrescar la sesión
El rol aparece como "admin" en vez de "administrador"Reiniciar el servidor (la migración DB actualiza roles antiguos)
El coordinador puede eliminar integrantesVerificar que el servidor esté actualizado (versión con validación de count)

Error al generar PDF de acta

  • Verificar que el navegador tenga permisos para popups
  • Si la imagen de fondo no se renderiza, usar Chrome o Edge

La base de datos se corrompió

  1. Detener el servidor
  2. Renombrar server/data/imjuve.db a imjuve.db.bak
  3. Reiniciar el servidor (creará una DB nueva)
  4. Ejecutar npx tsx src/seed.ts si se necesitan datos de prueba
Nota: Para reportar errores o solicitar soporte técnico, contactar al área de sistemas de IMJUVE.
